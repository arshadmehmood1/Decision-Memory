Quick Create Flow (30 seconds):
Step 1: Decision Title
- Input: Text field (120 char max)
- Placeholder: "e.g., Switch from Mailchimp to ConvertKit"

- Dropdown: Product, Marketing, Sales, Hiring, Tech, Operations, Strategic, Other

Standard Create Flow (3-5 minutes):
Includes Quick Create fields PLUS:
Context (Required)
- Rich text editor (2000 char max)
- Prompt: "Why are you making this decision right now?"
- Helper text: "Example: Our email open rates dropped 20% last month..."

Alternatives Considered (Required - Min 1)
- Add/remove alternative fields
- Each alternative:
  ‚Ä¢ Name: Text input
  ‚Ä¢ Why rejected: Text area (200 char)

- Add/remove assumption fields
- Text input per assumption
- Placeholder: "e.g., We'll send 50k emails/month within 6 months"

Success Criteria (Required - Min 1)
- Add/remove criteria fields
- Text input per criterion
- Placeholder: "e.g., Open rate improves by 10%"

Optional Fields (Expandable Section):
- Estimated Impact: Low | Medium | High | Critical
- Reversibility: Easy | Moderate | Hard | Irreversible
- Confidence Level: Slider 0-100%
- Timeline to Validate: Date picker
- Budget Impact: Number input ($)
- Tags: Comma-separated text input
Decision Creation Rules:
Must fill ALL required fields to save
Can save as "Draft" with only title
Draft decisions shown separately in dashboard
Max 5 alternatives, 10 assumptions, 5 success criteria

1. Query all decisions for workspace (min 20)
2. Calculate statistics:
   - Success rate (% of decisions marked Succeeded)
   - Failure rate by category
   - Assumption validation rates
   - Reversal patterns (same category, similar reasons)
3. Use GPT-4o to generate natural language insights
4. Store insights in DB (insights table)
5. Display on Insights page

Prompt Template for GPT-4o:
"Analyze these decision data points: [JSON of decisions]
Generate 3-5 actionable insights about patterns, blindspots, or trends.
Format as: Pattern + Evidence + Suggestion."

AI Integration Architecture
Embedding Generation:
// When decision created/updated
async function generateEmbedding(decision: Decision) {
  const text = `${decision.title} ${decision.context} ${decision.theDecision}`;
  
  const response = await openai.embeddings.create({
    model: "text-embedding-3-small",
    input: text,
  });
  
  const embedding = response.data[0].embedding;
  
  // Store embedding as JSON string in DB
  await prisma.decision.update({
    where: { id: decision.id },
    data: { embedding: JSON.stringify(embedding) }
  });
}
Similar Decision Search:
// Using pgvector extension in Postgres
async function findSimilarDecisions(decisionId: string, limit = 3) {
  const decision = await prisma.decision.findUnique({
    where: { id: decisionId },
    select: { embedding: true }
  });
  
  if (!decision?.embedding) return [];
  
  // Raw SQL query using pgvector cosine similarity
  const similar = await prisma.$queryRaw`
    SELECT id, title, status, "madeOn"
    FROM "Decision"
    WHERE embedding IS NOT NULL
      AND id != ${decisionId}
      AND "workspaceId" = (SELECT "workspaceId" FROM "Decision" WHERE id = ${decisionId})
    ORDER BY embedding <=> ${decision.embedding}::vector
    LIMIT ${limit}
  `;
  
  return similar;
}
AI Tagging:
async function generateTags(decision: Decision) {
  const prompt = `Analyze this decision and generate 3-5 relevant tags (single words or 2-word phrases):

Title: ${decision.title}
Context: ${decision.context}
Decision: ${decision.theDecision}

Generate tags that capture: topic, domain, strategy type.
Output as JSON array: ["tag1", "tag2", "tag3"]`;

  const response = await openai.chat.completions.create({
    model: "gpt-4o-mini",
    messages: [{ role: "user", content: prompt }],
    temperature: 0.3,
  });
  
  const tags = JSON.parse(response.choices[0].message.content || "[]");
  return tags.slice(0, 5); // Max 5 tags
}

// Background job runs weekly
async function generateInsights(workspaceId: string) {
  const decisions = await prisma.decision.findMany({
    where: { workspaceId },
    include: { assumptions: true, review: true }
  });
  
  if (decisions.length < 20) return; // Need min 20 decisions
  
  // Calculate stats
  const stats = {
    total: decisions.length,
    succeeded: decisions.filter(d => d.status === 'SUCCEEDED').length,
    failed: decisions.filter(d => d.status === 'FAILED').length,
    reversed: decisions.filter(d => d.status === 'REVERSED').length,
    byCategory: groupBy(decisions, 'category'),
    assumptionAccuracy: calculateAssumptionAccuracy(decisions),
  };
  
  // Use GPT to generate insights
  const prompt = `Analyze these decision patterns and generate 3-5 actionable insights:

Statistics:
${JSON.stringify(stats, null, 2)}

Sample failed decisions:
${decisions.filter(d => d.status === 'FAILED').slice(0, 5).map(d => ({
  title: d.title,
  category: d.category,
  review: d.review?.whatWentWrong
}))}

Generate insights in format:
{
  "insights": [
    {
      "type": "RECURRING_FAILURE" | "ASSUMPTION_BLINDSPOT" | "CATEGORY_TREND",
      "title": "Short title",
      "description": "Detailed insight with evidence and suggestion"
    }
  ]
}`;

  const response = await openai.chat.completions.create({
    model: "gpt-4o",
    messages: [{ role: "user", content: prompt }],
    response_format: { type: "json_object" },
  });
  
  const { insights } = JSON.parse(response.choices[0].message.content);
  
  // Store insights in DB
  for (const insight of insights) {
    await prisma.insight.create({
      data: {
        workspaceId,
        insightType: insight.type,
        title: insight.title,
        description: insight.description,
        relatedDecisions: [], // Could be enhanced to link specific decisions
      }
    });
  }
}

// Using Upstash Redis
import { Ratelimit } from "@upstash/ratelimit";

const ratelimit = new Ratelimit({
  redis: Redis.fromEnv(),
  limiter: Ratelimit.slidingWindow(100, "1 h"), // 100 requests per hour

export async function rateLimit(userId: string) {
  const { success } = await ratelimit.limit(userId);
